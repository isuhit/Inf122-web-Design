Module 3: Styling with CSS3

HTML provides the skeletal structure of a web application, but Cascading Style Sheets (CSS3) control the visual presentation. In this module, we explore how to isolate presentation rules from your structural data.

Unit 1: Anatomy of CSS & Selectors

CSS works by targeting HTML elements and applying property-value modifications to them. Styling can be applied via three methods:

  • Inline: Directly inside the HTML tag using the style attribute (not recommended for large projects).
  • Internal: Inside a <style> block in the document head (like this course portal page).
  • External: Linked as a separate .css file using a <link> tag (best practice for production).

Unit 2: The CSS Box Model

Every element rendered on a webpage is treated as a rectangular box. Understanding how these boxes calculate space is essential for preventing alignment bugs.

[Content Area]

Surrounded by Padding (inner space) → Border (the edge) → Margin (outer space).

  • Content: The actual text or image inside the element.
  • Padding: Clear space immediately surrounding the content (inside the border).
  • Border: A line encompassing the padding and content.
  • Margin: Clear space outside the border used to separate the element from its neighbors.

Unit 3: Modern Layout Layouts (Flexbox & Grid)

Legacy layouts relied heavily on table elements or CSS floats. Modern CSS introduces Flexbox (Flexible Box Layout) for fast, responsive one-dimensional alignments.
